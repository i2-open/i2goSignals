# goSignals v0.11.1

Released: 2026-06-07
Security patch on top of [v0.11.0](https://github.com/i2-open/i2goSignals/releases/tag/v0.11.0). No API or behavioral changes — drop-in upgrade.

## Why this release

Docker Hub vulnerability scanning flagged the published v0.11.0 image. A `govulncheck` audit confirmed three Go standard-library vulnerabilities (all reachable through the running server) and a stale base-image digest. This release rolls a clean Go toolchain build and refreshes the Chainguard base.

## Fixes

### Go standard-library CVEs (fixed by toolchain bump to 1.26.4)

| ID | Package | Reachable from |
|---|---|---|
| **GO-2026-5039** | `net/textproto` — arbitrary input in errors without escaping | `LoadKeyHandler` (key upload path) |
| **GO-2026-5037** | `crypto/x509` — quadratic candidate-hostname parsing | SPIFFE mTLS, OAuth TLS dial, token-list handler |
| **GO-2026-5038** | `mime.WordDecoder` — quadratic header decode | imported, not directly called |

### Base image
- Chainguard `cgr.dev/chainguard/bash:latest` digest refreshed from `4bfd053f…0a58` → `580c4bea…80e5` (multi-arch index, rebuilt 2026-06-07). Clears Docker Hub scanner warnings against the now-stale snapshot.

## Toolchain

- `go.mod` minimum Go version: `1.26.3` → `1.26.4`.
- CI (`.github/workflows/ci.yml`) and the dev image (`Dockerfile-dev`) bumped in step.

## Verification

- `govulncheck ./...` — **no vulnerabilities** under 1.26.4.
- `go test -race -count=1 ./...` — all packages pass.

## Upgrade

Drop-in. If you build from source, your local `go` must be ≥ 1.26.4 (or rely on Go's automatic toolchain download). No env-var, config, or API changes from v0.11.0.
