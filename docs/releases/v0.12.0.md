# goSignals v0.12.0

Released: TBD

## Why this release

v0.12.0 lays the persistence-layer groundwork for SSTP
(draft-hunt-secevent-sstp-00) as a third SET delivery method. The first piece —
idempotent SET ingestion (PRD #153) — is also a correctness fix for the existing
RFC 8935 push and RFC 8936 poll paths: a re-delivered SET is now a no-op at the
storage edge instead of double-counting `eventsIn` and re-fanning out to every
matching outbound stream. See ADR 0017 for the contract.

## Data-layer breaking changes

> **Breaking (data-layer):** `eventCol.jti` is now a sparse-unique index.
> Existing deployments with duplicate JTIs in the events collection will see a
> WARN log at startup and the index will not be created — the
> duplicate-suppression guarantee is OFF until the duplicates are cleaned up and
> the server is restarted. Suggested cleanup snippet (Mongo shell):
> `db.eventCol.aggregate([{$group:{_id:"$jti",ids:{$push:"$_id"},c:{$sum:1}}},{$match:{c:{$gt:1}}}])`
> then delete extras.

Memory-provider deployments are not affected at rest (the map only ever held
one entry per JTI), but the contract is now explicit: a duplicate insert
returns `interfaces.ErrDuplicateJTI` instead of silently overwriting.

## API surface: service layer is now public (PRD #175)

The service layer has been lifted from `internal/services` to **`pkg/services`**,
so an out-of-tree consumer can build a SET router on top of the stream, key,
event, server, token, client, and subject-filter services. The exported API is
unchanged from the old `internal/services` (history is preserved via `git mv`;
`git log --follow` traces every file back). Its transitive dependencies were made
public ahead of this lift: `pkg/dao` / `pkg/dao/ids` (#176, #177) and
`pkg/authSupport` (#178).

The file-backed in-memory DAO was also lifted from `internal/dao/memory` to
**`pkg/dao/memory`** — a deliberate deviation from PRD #175's original intent to
keep it internal — so it can back the now-public service tests and serve as a
Mongo-free test double for downstream consumers. The Mongo DAO
(`internal/dao/mongo`) stays internal. A scoped CI gate now fails the build if
`pkg/services`, `pkg/dao`, or `pkg/authSupport` ever import `internal/*`. See
ADR 0021 for the full rationale and the recorded deviation.

If you imported `github.com/i2-open/i2goSignals/internal/services` or
`…/internal/dao/memory`, update the path to `…/pkg/services` /
`…/pkg/dao/memory`; no symbol or signature changes are required.
