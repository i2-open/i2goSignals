# goSignals v0.11.0

Released: 2026-06-XX
Compared against [v0.10.0](https://github.com/i2-open/i2goSignals/releases/tag/v0.10.0) (2026-05-02). 142 commits, ~25 PRs.

## Highlights

- **Subject filtering** — full multi-mode subject filter pipeline (LOCAL, POLL, HYBRID, PASSTHRU) with admin review and a §9 removal grace period.
- **OAuth delegated login + token administration** — CLI can sign in via Keycloak; server gains RFC 7662 introspection and RFC 7009 revocation, redemption tracking, and inbound bearer hardening (audience + algorithm allow-list).
- **Push recovery state machine** — T1 reactive / T2 pre-flight / T3 idle-keepalive replaces the implicit backfill loop, with Prometheus metrics for every transition.
- **Environment variable taxonomy** — all `I2SIG_*` vars renamed under a single v0.11.0 scheme; legacy names continue to work via `envcompat` and emit a deprecation warning.
- **Observability** — Grafana over TLS with Keycloak SSO, Prometheus and Loki on TLS, the Alloy log pipeline, and dashboards covering the push state machine and i2scim peers.
- **Go 1.26.2** toolchain and the first GitHub Actions CI workflow (build / vet / `go test -race`).

## New features

### Subject filtering (PRD #89, #97, #106)
- Pure subject identity + matching module (#91) underpins every mode.
- Discovery gating and stream-config fields exposed in well-known metadata (#90).
- **LOCAL** subject filtering on single-node push streams (#92).
- **POLL** delivery-time filtering (#93).
- **HYBRID** refcounted upstream relay (#96) and **PASSTHRU** relay with target validation (#95).
- Cluster cache invalidation on filter changes (#94).
- Admin review surface + §9 removal grace period (#105).
- CLI `get`/`set` verbs gain subject-filter manipulation (#113).
- `event_source.type` validation enforced in stream create/update (#135); review now exposes the effective `event_source` and grace status, with `nil` resolving to `AUDIENCE` (#118).

### Token administration — PRD #128
- `GET /admin/tokens` — list with filters.
- `POST /admin/introspect` — RFC 7662.
- `POST /admin/revoke` — RFC 7009.
- Redemption tracking and automatic expiry of unused registrations.

### OAuth delegated login + CLI bootstrap — PRD #120
- goSignals CLI logs in via Keycloak (delegated OAuth) and persists session credentials safely for unattended bootstrap (#127).
- Management commands migrated to `serverBearer` for OAuth-logged-in users (#138).
- Zero `Expiry` treated as "needs refresh" rather than never-expiring (#142).
- `GOSIGNALS_HOME` honored when it points at a directory (#143).

### SSF transmitter CLI — PRD #83 (Keycloak demo, partial)
- `client-credentials` and `poll-receive --tx-alias` (#134). Keycloak-side slices remain on hold pending the upstream native-SSF release.

### Push recovery state machine
- `recoveryLoop` with T1 reactive, T2 pre-flight, and T3 idle-keepalive verify-event generator.
- `updateStream(state, reason)` transition helper centralises lifecycle changes.
- `goSetPush` adds a `FailureClass` classifier and Retry-After parsing.
- Prometheus instrumentation across the push state machine (`I2SIG_PUSH_*` documented in `docs/Metrics.md`).
- `docs/operations.md` operator guide added.

### Poll delivery
- Configurable default and maximum long-poll timeout (#62).
- Poll-receiver dead-goroutine revival and bounded 403 retries (#119).

### Observability
- Grafana served over TLS with Keycloak SSO across all compose stacks (#76).
- Prometheus scrape and Loki + Alloy log pipeline on TLS.
- i2scim peers wired into the Loki/Prometheus stack (#73).
- Startup-notice dashboard provisioned out of the box.
- `genTlsKeys` becomes SAN-aware for server cert regeneration.
- Structured JSON logging with default service-identity attributes; sub-logger handlers bind on first use so `package init()` calls to `logger.Sub()` pick up `Init()`.
- Log-level policy documented (`WARN` for auth failures and transient stream failures; `ERROR` reserved for human-intervention conditions).

### Build, release, CI
- GitHub Actions workflow for build / vet / `go test -race` on Go 1.26.2.
- Single-source versioning — `pkg/constants/version.txt` drives binary, image, and release tag; `build.sh` folded into the Makefile.
- Multi-arch image now publishes a single manifest with `:latest`; `make build-docker-multiarch` is push-opt-in via `PUSH=1`.
- Image bundles `THIRD-PARTY-NOTICES`; `go-licenses` check added to the toolchain.
- Copyright notice printed on startup.

## Breaking and behavioral changes

### Environment variable rename (PR #72, follow-ups)
All server env vars have been migrated under the v0.11.0 taxonomy in these groups: `STREAM_*`, `ISSUER_*`, `AUTH_*`, `CLUSTER_*`, `STORE_MONGO_*`, `STORE_MEM_*`, `PUSH_*`, `POLL_*`, `TLS_*`, `SPIFFE_*`. Legacy names continue to work via `internal/envcompat` and log a deprecation warning at startup. `docs/configuration_properties.md` is rewritten under the new taxonomy. **Action:** review your compose/.env files at upgrade and migrate before the next minor release.

### OAuth bearer hardening (#144, #146)
- Inbound OAuth bearers must now match the server's audience and an explicit algorithm allow-list (asymmetric-only). Symmetric algorithms are rejected.
- The implicit bare-root role has been dropped — issue scoped tokens explicitly.
- Server type is now determined from the well-known `gosignals_version` rather than heuristics (#141).
- See `docs/security_model.md` for the full reconciled model.

### Authorization fixes
- `tx_alias` creation, `StreamUpdate`, and key-material guards now honour OAuth/STS scope correctly (#150).
- Token-admin endpoints honour OAuth/STS admin scope (#128).
- Foreign-server provisioning requires admin scope (#139).
- Registration rejects an all-dropped explicit scope list with 400 (#137).
- Key-scope callers cannot upload key material (#127 follow-up).
- Non-project-bound tokens may update streams by `stream_id` (#114).

### Other
- `EventRecord` renamed to `AgEventRecord` across the codebase. Downstream callers that imported the type need to rename.

## Refactors (internal, no API change)

A long-running cleanup of the persistence layer landed across issues #40–#47 and #51–#53:

- Receiver-stream predicate lifted from DAOs into `streamService` (#40).
- String IDs across persistence interfaces (#41).
- `ClusterCoordinator` and `Storage` seams extracted (#42).
- Rebindable Mongo collections via `atomic.Pointer` (#46).
- `tx_alias` resolution and `IssuerJWKSUrl` normalisation lifted into the service layer (#43).
- Consumers depend on services directly; `MongoProvider` wrapper-method tax removed (#47).
- Memory provider `WriteHook` replaced with a DAO-level notifying decorator (#44).
- `BaseProvider` and `DbProviderInterface` deleted; consumers depend on a `Persistence` record (#45).
- Server handler tree moved under `internal/server` (#51).
- `StreamEventMatch` lifted into `EventService.MatchesStream` (#52).
- `PushDelivery` seam introduced with HTTP and in-memory adapters (#53).

## Upgrade notes

1. **Env vars:** legacy names still work but warn. Plan to rename in your deployment manifests before v0.12.
2. **OAuth audience/alg:** if you issue HS256 (or other symmetric) bearers to goSignals, switch to RS/ES/EdDSA before upgrading.
3. **Bare-root role removed:** issue explicit scopes to any caller that relied on root.
4. **`AgEventRecord` rename:** rebuild any external Go consumers of the type.
5. **Go toolchain:** the project now requires Go 1.26.2 for `go install` builds.
