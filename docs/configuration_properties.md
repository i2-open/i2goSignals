# i2goSignals Server Configuration Properties

Currently i2goSignals has minimal configuration properties. The current values are:

| Area / Parameter             | Description                                                                                                                                | Default                                                |
|------------------------------|--------------------------------------------------------------------------------------------------------------------------------------------|--------------------------------------------------------|
| **Server**                   |                                                                                                                                            |                                                        |
| `PORT`                       | The port number on which goSignals will serve requests.                                                                                    | 8888                                                   |
| `BASE_URL`                   | The host and port of the server. e.g. 127.0.0.1:8888                                                                                       | 127.0.0.1:<PORT> or 127.0.0.1:8888                     |
| `LOG_LEVEL`                  | The logging level for the server. Possible values: `debug`, `info`, `warn`, `error`.                                                       | `info`                                                 |
| <BR>**Identity**             |                                                                                                                                            |                                                        |
| `I2SIG_ISSUER`               | The default issuer identifier for the server.                                                                                              | `DEFAULT`                                              |
| `I2SIG_TOKEN_ISSUER`         | The issuer identifier used for tokens generated by this server.                                                                            | `DEFAULT`                                              |
| <BR>**Authentication**       |                                                                                                                                            |                                                        |
| `OAUTH_SERVERS`              | Comma-separated list of OAuth/OIDC discovery URLs (e.g., Keycloak) used to validate external Bearer tokens.                                | _none_                                                 |
| <BR>**Clustering**           |                                                                                                                                            |                                                        |
| `NODE_ID`                    | Unique identifier for the server node in a cluster.                                                                                        | `POD_NAME` if set, else `hostname-timestamp`           |
| `POD_NAME`                   | Fallback for `NODE_ID` when running in Kubernetes.                                                                                         | _none_                                                 |
| <BR>**Mongo DB**             |                                                                                                                                            |                                                        |
| `MONGO_URL`                  | The connection URL used to connect to a Mongo DB. Note: Should be a clustered replica set.                                                 | mongodb://root:dockTest@0.0.0.0:8880<br>_testing only_ |
| `DBNAME`                     | The name of the database to store goSignals data.                                                                                          | ssef                                                   |
| `MONGO_WATCH_FILE`           | Path to the file where MongoDB resume tokens are stored.                                                                                   | `resources/mongo_token.json`                           |
| `MONGO_FAILTOMEM`            | If FALSE, the server will fail to start if it cannot connect to the database.                                                              | true                                                   |
| <BR>**Memory Provider**      |                                                                                                                                            |                                                        |
| `MEM_DIRECTORY`              | Directory to store persistent state for the memory provider.                                                                               | `config/{dbName}`                                      |
| `MEM_SAVE_RATE`              | The interval in seconds between periodic saves. A setting of 0 means every change.                                                        | 30                                                     |
| <BR>**Roles**                |                                                                                                                                            |                                                        |
| `SSEF_ADMIN_ROLE`            | The role to use to identify clients with administrative rights within a project.                                                           | ADMIN                                                  |
| <BR>**Poll Receiver**        |                                                                                                                                            |                                                        |
| `POLL_STATUS_CHECK_INTERVAL` | The interval in seconds to re-check transmitter status when the stream is paused.                                                          | 30                                                     |
| `POLL_RETRY_BASE_DELAY`      | The base delay in seconds for exponential backoff during polling retries.                                                                  | 1.0                                                    |
| `POLL_RETRY_MAX_DELAY`       | The maximum delay in seconds for exponential backoff during polling retries.                                                               | 300.0                                                  |
| `POLL_RETRY_BACKOFF_FACTOR`  | The factor by which the delay increases during exponential backoff.                                                                        | 2.0                                                    |
| `POLL_RETRY_LIMIT`           | The maximum time in seconds to continue retrying before disabling the stream.                                                              | 21600 (6 hours)                                        |
| `POLL_SRV_BEHAVIOR`         | Polling behavior based on status. `MODE` (default) follows stream status (403/503). `ALWAYSON` ignores disabled/paused status for polling. | `MODE`                                                 |
| <BR>**TLS Config**           | <BR>To be implemented!                                                                                                                     |                                                        |



For the goSignals tool the following properties are defined:

| Area / Parameter    | Description                                                                                | Default                  |
|---------------------|--------------------------------------------------------------------------------------------|--------------------------|
| `GOSIGNALS_HOME`    | File path to store local administration configuration data                                 | ~/.goSignals/config.json |
| `GOSIGNALS_SCRIPT`  | Path to a script file containing goSignals commands to be executed on startup.             | _none_                   |
| `LOG_LEVEL`         | The logging level for the tool.                                                            | `info`                   |
